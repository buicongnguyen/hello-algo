02 · Data-structure field guide

Choose the shape that makes the operation cheap

The structure is not decoration—it decides which work is direct, which work requires a walk, and how values sit in memory.

Memory shapeContiguous slots

The address of an element follows directly from the base address and its index.

Direct accessAccess · O(1)

Array

Values occupy neighboring memory slots, so an index can jump straight to its element. Moving later elements makes middle insertion expensive.

Reach for it when
You need fast indexed reads and cache-friendly iteration.
Watch for
Middle insertions, deletions, and resizing can move many elements.

05 · Binary-search lab

Throw away half, then repeat

Binary search works because sorted data tells you which half cannot contain the target. Follow the interval until the value is found—or the interval becomes empty.

lowsearch intervalhigh
ReadySearch for 42

The full sorted array is the first candidate interval.

low0
mid—
high11
Loop invariant

If the target exists, it is still inside the highlighted interval.

Why it is fastO(log n)

One comparison removes roughly half of the candidates.

What it needsSorted + indexable

Jumping to the middle is efficient in arrays, not linked lists.

When not to use itSmall or changing data

Sorting and maintaining order can cost more than a direct scan.

08 · Four problem-solving patterns

Similar recursion trees, different promises

All four strategies reduce the remaining problem. What changes is whether branches are independent, repeated, reversible, or safely decided one step at a time.

Independent subproblems

Divide & conquer

Divide→Solve parts→Combine

Use when the smaller problems do not overlap and their answers can be merged.

Think of
Merge sort · binary search · tree construction
Proof question
Do solved parts reconstruct the whole?
Read chapter →
Reversible choices

Backtracking

Choose→Explore→Undo

Depth-first search through a choice space; constraints prune branches that cannot succeed.

Think of
Permutations · subset sum · N-Queens
Speed question
What fact proves this branch is impossible?
Read chapter →
Overlapping subproblems

Dynamic programming

State→Transition→Store

Memoization works top-down; tabulation fills states bottom-up. Both avoid solving the same state twice.

Think of
Knapsack · coin change · edit distance
Design question
What minimum state fully determines the future?

Irreversible local choice

Greedy

Choose best now→Reduce→Prove

Fast and elegant only when a local choice can be proven to preserve a global optimum.

Think of
Fractional knapsack · interval choices
Proof question
Can an optimal solution exchange in this choice?
