# 자료구조와 알고리즘이란?

## 자료구조

자료구조(data structure)는 데이터를 조직하고 저장하는 방식입니다. 배열은 값을 연속된 공간에 두어 인덱스로 빠르게 접근하고, 연결 리스트는 노드를 참조로 이어 유연하게 삽입합니다. 스택, 큐, 해시 테이블, 트리, 그래프는 각각 다른 관계와 접근 순서를 표현합니다.

자료구조를 선택할 때는 다음 질문이 중요합니다.

- 어떤 연산을 가장 자주 수행하는가?
- 데이터의 크기와 모양은 어떻게 변하는가?
- 순서, 중복, 계층, 연결 관계를 보존해야 하는가?
- 허용할 수 있는 시간과 메모리 비용은 얼마인가?

## 알고리즘

알고리즘(algorithm)은 입력을 받아 유한한 단계로 처리하고 원하는 출력을 만드는 명확한 절차입니다. 좋은 알고리즘은 다음 특성을 가집니다.

1. 각 단계가 모호하지 않습니다.
2. 유효한 입력에 대해 종료합니다.
3. 요구한 결과를 정확하게 만듭니다.
4. 시간과 공간을 합리적으로 사용합니다.

예를 들어 정렬된 배열의 목표값을 찾을 때 선형 탐색은 앞에서부터 하나씩 확인하고, 이진 탐색은 중앙값과 비교해 절반을 제외합니다. 두 방법은 같은 답을 만들 수 있지만 입력이 커질 때 필요한 작업량은 크게 다릅니다.

```python
def binary_search(nums: list[int], target: int) -> int:
    left, right = 0, len(nums) - 1
    while left <= right:
        middle = (left + right) // 2
        if nums[middle] == target:
            return middle
        if nums[middle] < target:
            left = middle + 1
        else:
            right = middle - 1
    return -1
```

자료구조와 알고리즘은 함께 설계해야 합니다. 데이터의 구조가 가능한 연산을 결정하고, 알고리즘은 그 구조의 장점을 활용합니다.
