# 정렬 알고리즘

정렬 알고리즘은 시간·공간 복잡도, 안정성, 제자리 수행 여부, 거의 정렬된 데이터에 대한 적응성으로 평가합니다.

![정렬 예제](sorting_algorithm.assets/sorting_examples.png)

## 주요 기준

- **안정성**: 같은 키를 가진 원소의 상대 순서를 보존합니다.
- **제자리 정렬**: $O(1)$ 또는 매우 적은 보조 공간을 사용합니다.
- **적응성**: 데이터가 거의 정렬되어 있을 때 더 빨라집니다.
- **비교 기반**: 두 원소의 비교 결과로 순서를 결정합니다.

모든 상황에서 가장 좋은 알고리즘은 없습니다. 입력 크기, 메모리 제한, 키 분포, 안정성 요구를 함께 고려해야 합니다.
