# 공간 복잡도

공간 복잡도는 알고리즘 실행에 필요한 메모리가 입력 크기 $n$에 따라 어떻게 증가하는지 나타냅니다. 분석 범위를 분명히 하기 위해 입력 자체의 저장 공간과 알고리즘이 추가로 사용하는 **보조 공간**을 구분하기도 합니다.

## 메모리는 어디에 사용되나요?

- 상수, 변수, 객체 같은 데이터
- 배열, 해시 테이블 같은 보조 자료구조
- 함수 호출마다 만들어지는 스택 프레임
- 반환해야 하는 출력 결과

```python
def reverse_copy(nums: list[int]) -> list[int]:
    result = []
    for index in range(len(nums) - 1, -1, -1):
        result.append(nums[index])
    return result
```

`result`는 입력 크기에 비례하므로 위 함수의 추가 공간은 $O(n)$입니다.

## 제자리 알고리즘

입력 구조를 직접 수정하고 입력 크기와 무관한 추가 변수만 쓰는 알고리즘을 흔히 제자리(in-place) 알고리즘이라고 합니다.

```python
def reverse_in_place(nums: list[int]) -> None:
    left, right = 0, len(nums) - 1
    while left < right:
        nums[left], nums[right] = nums[right], nums[left]
        left += 1
        right -= 1
```

이 구현의 보조 공간은 $O(1)$입니다. 그러나 입력을 변경하므로 호출자가 원본을 보존해야 한다면 적합하지 않을 수 있습니다.

## 재귀와 호출 스택

재귀 깊이가 $n$이면 동시에 유지되는 스택 프레임도 최대 $n$개이므로 보통 $O(n)$ 공간이 필요합니다. 분기 수가 많아 전체 호출 수가 지수적이어도 깊이 우선으로 실행한다면 동시에 존재하는 프레임 수는 재귀 깊이에 의해 결정됩니다.

## 시간–공간 트레이드오프

계산 결과를 해시 테이블이나 배열에 저장하면 메모리를 더 쓰는 대신 반복 계산을 줄일 수 있습니다. 반대로 값을 저장하지 않고 다시 계산하면 공간은 줄지만 시간이 늘어납니다. 실제 설계에서는 입력 규모, 메모리 한도, 응답 시간, 코드 명확성을 함께 비교해야 합니다.
