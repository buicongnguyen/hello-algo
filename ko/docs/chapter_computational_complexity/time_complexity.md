# 시간 복잡도

시간 복잡도는 입력 크기 $n$이 증가할 때 알고리즘의 기본 연산 횟수가 어떻게 증가하는지 나타냅니다. 정확한 초가 아니라 증가율을 비교합니다.

## 점근 표기와 빅오

빅오 표기법 $O(f(n))$은 입력이 충분히 클 때 작업량의 점근적 상한을 표현합니다. 증가율에 큰 영향을 주지 않는 상수 계수와 낮은 차수 항은 생략합니다.

$$
3n^2 + 2n + 1 = O(n^2)
$$

## 대표적인 시간 복잡도

| 종류 | 표기 | 예시 |
| --- | --- | --- |
| 상수 | $O(1)$ | 배열의 인덱스 접근 |
| 로그 | $O(\log n)$ | 정렬된 배열의 이진 탐색 |
| 선형 | $O(n)$ | 모든 항목 한 번 순회 |
| 선형 로그 | $O(n \log n)$ | 효율적인 비교 정렬 |
| 이차 | $O(n^2)$ | 모든 항목 쌍 비교 |
| 지수 | $O(2^n)$ | 가지치기 없는 일부 선택 탐색 |
| 팩토리얼 | $O(n!)$ | 모든 순열 생성 |

```python
def constant(nums: list[int]) -> int:
    return nums[0]

def linear(nums: list[int]) -> int:
    total = 0
    for value in nums:
        total += value
    return total

def quadratic(nums: list[int]) -> int:
    count = 0
    for left in nums:
        for right in nums:
            if left < right:
                count += 1
    return count
```

## 반복문의 분석

연속된 코드 블록의 복잡도는 가장 빠르게 증가하는 항이 지배합니다. 중첩 반복문은 각 반복 횟수를 곱해 분석합니다. 단, 반복 범위가 이전 인덱스에 따라 달라지면 실제 합을 확인해야 합니다.

## 재귀의 분석

재귀는 호출 횟수와 각 호출의 작업량을 함께 봅니다. 이진 탐색은 문제 크기를 절반으로 줄이므로 깊이가 $O(\log n)$입니다. 병합 정렬은 $\log n$개의 층에서 각 층마다 전체 $n$개 항목을 처리하므로 $O(n \log n)$입니다.

## 최선·최악·평균적인 경우

선형 탐색에서 첫 항목이 목표라면 $O(1)$이지만 목표가 마지막에 있거나 없다면 $O(n)$입니다. 빅오를 말할 때 어떤 경우를 분석하는지 분명히 해야 합니다. 일반적으로 보장해야 할 상한에는 최악의 경우를 사용합니다.
