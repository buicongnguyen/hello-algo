# 2장 요약

- 실제 실행 시간은 환경에 의존하므로 복잡도 분석은 입력 크기에 따른 자원 증가율을 비교합니다.
- 빅오 표기법은 입력이 충분히 클 때 작업량의 점근적 상한을 표현합니다.
- 대표적인 증가율은 $O(1)$, $O(\log n)$, $O(n)$, $O(n \log n)$, $O(n^2)$, $O(2^n)$, $O(n!)$입니다.
- 반복문은 실행 횟수와 중첩 관계를, 재귀는 호출 수와 각 호출의 작업량을 분석합니다.
- 재귀의 공간 비용은 전체 호출 수가 아니라 동시에 유지되는 최대 호출 깊이와 관련됩니다.
- 공간 복잡도는 보조 자료구조, 출력, 호출 스택 등 실행 중 필요한 메모리의 증가율을 나타냅니다.
- 제자리 알고리즘은 보조 공간을 줄이지만 입력을 변경할 수 있습니다.
- 캐싱과 메모이제이션은 공간을 더 사용해 실행 시간을 줄이는 대표적인 시간–공간 트레이드오프입니다.
- 최선, 최악, 평균적인 경우 중 무엇을 분석했는지와 그 가정을 명확히 밝혀야 합니다.
