# 반복과 재귀

반복(iteration)과 재귀(recursion)는 작업을 되풀이하는 두 가지 기본 방법입니다.

## 반복

반복문은 조건이 만족되는 동안 같은 코드 블록을 실행합니다.

```python
def for_loop(n: int) -> int:
    result = 0
    for value in range(1, n + 1):
        result += value
    return result
```

반복은 상태 변화를 코드에서 직접 볼 수 있고 보통 추가 호출 스택이 필요하지 않습니다.

## 재귀

재귀 함수는 더 작은 같은 문제를 호출합니다. 올바른 재귀에는 두 요소가 필요합니다.

1. 더 이상 재귀하지 않는 **기저 조건**
2. 입력을 기저 조건에 가까워지게 만드는 재귀 단계

```python
def recur(n: int) -> int:
    if n == 1:
        return 1
    return n + recur(n - 1)
```

호출할 때마다 매개변수와 지역 변수, 돌아갈 위치가 호출 스택의 스택 프레임에 저장됩니다. 재귀 깊이가 너무 크면 스택 공간이 부족해질 수 있습니다.

## 재귀 트리

한 호출이 여러 호출로 갈라지면 재귀 트리로 작업량을 볼 수 있습니다. 각 노드는 함수 호출이고 간선은 하위 호출입니다. 트리의 노드 수는 시간 비용, 높이는 최대 호출 스택 깊이와 관련됩니다.

```python
def fibonacci(n: int) -> int:
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
```

위 단순 피보나치 구현은 같은 부분 문제를 반복 계산합니다. 메모이제이션이나 동적 계획법으로 이미 계산한 값을 저장하면 중복 작업을 제거할 수 있습니다.

반복과 재귀 중 하나가 항상 더 좋은 것은 아닙니다. 문제 구조, 언어의 스택 제한, 코드의 명확성, 필요한 공간을 함께 고려합니다.
