# 알고리즘 효율성 평가

## 실제 실행으로 측정하기

같은 입력으로 여러 구현을 실행하고 시간을 재면 현실적인 결과를 얻을 수 있습니다. 그러나 이 방법만으로 일반적인 효율성을 판단하기는 어렵습니다.

- 하드웨어와 운영체제의 영향을 받습니다.
- 프로그래밍 언어와 컴파일러 최적화가 결과를 바꿉니다.
- 작은 입력에서는 증가율 차이가 잘 보이지 않습니다.
- 가능한 모든 입력을 시험할 수 없습니다.

## 입력 크기로 비교하기

복잡도 분석에서는 입력 크기를 $n$으로 두고 기본 연산 횟수의 증가 추세를 봅니다. 예를 들어 한 번 순회하는 코드는 대략 $n$번, 중첩된 두 반복문은 대략 $n^2$번 작업합니다.

```python
def linear_sum(nums: list[int]) -> int:
    total = 0
    for value in nums:
        total += value
    return total

def count_pairs(nums: list[int]) -> int:
    count = 0
    for _ in nums:
        for _ in nums:
            count += 1
    return count
```

$n$이 두 배가 되면 선형 작업은 약 두 배, 이차 작업은 약 네 배가 됩니다. 이 차이는 작은 입력보다 큰 입력에서 훨씬 중요합니다.

## 최악의 경우를 보는 이유

입력에 따라 실행 횟수가 달라질 수 있습니다. 최악의 경우는 성능의 상한을 제공하므로 시스템이 감당해야 할 최대 비용을 설명하기 쉽습니다. 필요하면 최선과 평균적인 경우도 함께 분석하되, 각각의 가정과 확률 모델을 분명히 해야 합니다.
