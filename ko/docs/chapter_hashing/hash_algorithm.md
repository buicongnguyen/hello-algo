# 해시 알고리즘

해시 알고리즘은 길이가 임의인 입력을 고정되거나 유한한 범위의 값으로 매핑합니다. 입력 범위가 출력 범위보다 크므로 여러 입력이 같은 결과를 낼 수밖에 없습니다.

![좋은 해시 분포와 나쁜 해시 분포](hash_algorithm.assets/hash_collision_best_worst_condition.png)

## 해시 알고리즘의 목표

용도에 따라 좋은 해시 함수에는 여러 성질이 필요합니다.

- **결정성**: 같은 규칙 안에서 같은 입력은 항상 같은 출력을 냅니다.
- **균등 분포**: 키가 버킷 전체에 비교적 고르게 퍼집니다.
- **효율성**: 지원하려는 연산보다 충분히 빠르게 계산됩니다.
- **눈사태 효과**: 입력의 작은 변화가 출력의 많은 비트를 바꿉니다.
- 암호학적 함수에서는 **역산과 의도적인 충돌 생성이 어려워야 합니다.**

조회 테이블용 해시 함수가 비밀번호 저장에도 적합한 것은 아닙니다. 비밀번호에는 salt를 사용하고 대량 추측 공격을 막도록 느리게 설계된 Argon2, scrypt, bcrypt 같은 전용 알고리즘이 필요합니다.

## 해시 알고리즘 설계

기본적인 설계는 다음 기법을 조합합니다.

- 상수와 더하거나 곱해 문자 위치를 혼합합니다.
- XOR와 비트 이동으로 변화를 확산합니다.
- 소수 또는 모듈로로 반복 패턴을 줄입니다.
- 정해진 순서로 각 바이트를 처리합니다.

다음 교육용 예제는 보안 목적으로 사용하면 안 됩니다.

```python
def simple_hash(text: str, capacity: int) -> int:
    value = 0
    for byte in text.encode("utf-8"):
        value = (value * 31 + byte) % capacity
    return value
```

Python은 일부 데이터형의 `hash()` 값을 프로세스마다 무작위화하여 충돌 공격을 줄입니다. 따라서 내장 `hash()` 결과를 파일에 저장하고 다음 실행에서도 같을 것이라 기대하면 안 됩니다.

## 대표적인 해시 알고리즘

- MD5와 SHA-1은 역사적으로 중요하지만 암호학적 충돌 안전성이 더 이상 충분하지 않습니다.
- SHA-2와 SHA-3 계열은 무결성 검사와 여러 프로토콜에 사용하는 현대적인 암호학적 해시입니다.
- MurmurHash, xxHash 같은 비암호학적 함수는 해시 테이블, 파티셔닝, 데이터 처리에서 속도와 분포를 우선합니다.

속도만 보지 말고 위협 모델과 호환성 요구에 따라 알고리즘을 선택해야 합니다.

## 자료구조의 해시 값

해시 테이블의 키는 동등성과 해시 사이에 다음 일관성을 지켜야 합니다.

$$
a = b \Rightarrow hash(a) = hash(b)
$$

반대 방향은 성립하지 않습니다. 서로 다른 두 키가 충돌할 수 있기 때문입니다. 키는 테이블에 들어 있는 동안 불변이어야 합니다. 해시를 결정하는 내용이 바뀌면 이전 버킷에 있는 키를 다시 찾지 못할 수 있습니다.

복합 객체는 보통 각 필드의 해시를 결합합니다. 동등성 비교에 참여하는 필드만 사용하고 결합 순서를 안정적으로 유지해야 합니다.
