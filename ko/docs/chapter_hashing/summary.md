# 6장 요약

### 핵심 복습

- 해시 테이블은 키–값 쌍을 저장하고 해시 함수로 버킷을 선택합니다.
- 삽입, 삭제, 검색의 평균 비용은 $O(1)$이지만 심한 충돌에서는 $O(n)$까지 느려질 수 있습니다.
- 적재율은 원소 수를 용량으로 나눈 값이며 적재율이 높으면 일반적으로 충돌이 늘어납니다.
- 확장할 때 더 큰 테이블을 할당하고 모든 키를 재해싱해야 합니다.
- 분리 연결법은 버킷마다 여러 원소를 저장하고 개방 주소법은 같은 배열의 다른 위치를 탐색합니다.
- 선형 탐사는 단순하지만 군집을 만들고 제곱 탐사와 이중 해싱은 탐사 순서를 바꿉니다.
- 좋은 해시 함수는 결정적이고 빠르며 고르게 분포해야 합니다. 암호학적 함수는 역산과 의도적 충돌에도 강해야 합니다.
- 같은 키는 같은 해시 값을 가져야 하고 테이블 안의 키는 불변이어야 합니다.
- 자료구조용 해시 함수는 비밀번호 전용 해시 알고리즘을 대신하지 못합니다.

### 질문과 답변

**충돌이 생기면 해시 함수가 잘못된 것인가요?**

아닙니다. 키 범위가 결과 수보다 크면 충돌은 필연적입니다. 중요한 것은 균등한 분포와 올바른 충돌 처리입니다.

**용량을 늘릴 때 왜 재해싱해야 하나요?**

인덱스는 보통 `capacity`에 의존합니다. 용량이 바뀌면 같은 해시 값도 다른 버킷에 매핑될 수 있습니다.

**Python의 `hash()`로 파일 무결성을 검사해도 되나요?**

안 됩니다. 프로세스마다 값이 달라질 수 있고 필요한 암호학적 특성도 없습니다. 표준 라이브러리의 SHA-256 같은 적절한 알고리즘을 사용하세요.

**변경 가능한 키가 조회를 망가뜨리는 이유는 무엇인가요?**

해시에 참여하는 필드가 바뀌면 새 해시는 다른 버킷을 가리키지만 객체는 여전히 이전 위치에 남아 있기 때문입니다.
