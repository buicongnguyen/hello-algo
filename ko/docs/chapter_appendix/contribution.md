# 함께 기여하기

프로젝트는 기술 오류 수정, 문장 개선, 예제 추가, 번역 검토를 환영합니다. 큰 변경을 시작하기 전 기여 지침을 읽고 issue에서 범위를 논의하세요.

## 작은 내용 수정

GitHub의 각 원문 페이지는 편집 버튼으로 열 수 있습니다. fork를 만들고 별도 브랜치에서 Markdown을 수정한 뒤 링크를 검사하고 변경 이유가 분명한 pull request를 보냅니다.

![페이지 편집 버튼](contribution.assets/edit_markdown.png)

## 콘텐츠 작성

- 용어집과 일관된 표현을 사용합니다.
- 출처를 밝히고 라이선스 정보를 제거하지 않습니다.
- 짧고 독립 실행 가능한 Python 예제를 우선합니다.
- 이미지, 수식, 링크, 빌드를 제출 전에 검사합니다.
- 관련 없는 변경은 별도 pull request로 나눕니다.

파생 콘텐츠는 계속 CC BY-NC-SA 4.0으로 공유되며 원본 프로젝트의 공식 후원을 의미하지 않습니다.
