# 자료구조의 분류

자주 사용하는 자료구조에는 배열, 연결 리스트, 스택, 큐, 해시 테이블, 트리, 힙, 그래프가 있습니다. 이들은 **논리 구조**와 **물리 구조**라는 두 관점으로 분류할 수 있습니다.

## 논리 구조: 선형과 비선형

**논리 구조는 데이터 원소 사이의 논리적 관계를 설명합니다.** 배열과 연결 리스트에서는 데이터가 일정한 순서로 놓여 선형 관계를 이룹니다. 트리는 데이터를 위에서 아래로 계층화하여 부모–자식 관계를 표현합니다. 그래프는 정점과 간선으로 복잡한 네트워크 관계를 나타냅니다.

논리 구조는 보통 두 가지 큰 범주로 나뉩니다.

- **선형 자료구조**: 배열, 연결 리스트, 스택, 큐처럼 원소가 일대일 순차 관계를 이룹니다.
- **비선형 자료구조**: 트리, 힙, 그래프처럼 원소가 하나의 연속된 줄에만 놓이지 않습니다. 충돌 처리 방식에 따라 해시 테이블은 선형 구조와 비선형 구조를 함께 사용할 수 있습니다.

비선형 자료구조는 다시 다음과 같이 나눌 수 있습니다.

- **트리형 구조**: 트리와 힙처럼 한 원소가 여러 자식 원소와 관계를 맺을 수 있습니다.
- **네트워크형 구조**: 그래프처럼 원소 사이에 다대다 관계가 존재합니다.

## 물리 구조: 연속 저장과 분산 저장

**프로그램이 실행되는 동안 처리 중인 데이터는 주로 메모리에 놓입니다.** 메모리를 매우 큰 스프레드시트라고 생각해 볼 수 있습니다. 각 칸에는 일정량의 데이터가 들어가고 고유한 주소가 붙습니다.

컴퓨터는 메모리 주소를 이용해 필요한 위치의 데이터를 찾습니다. 프로그램은 이 주소를 통해 메모리의 값을 읽거나 갱신합니다.

![RAM, 메모리 공간, 메모리 주소](classification_of_data_structure.assets/computer_memory_location.png)

> 메모리를 스프레드시트에 비유한 것은 단순한 설명입니다. 실제 동작에는 주소 공간, 메모리 관리, 캐시, 가상 메모리, 물리 메모리 같은 요소가 함께 관여합니다.

메모리는 여러 프로그램이 공유하는 자원입니다. 어떤 메모리 영역을 한 프로그램이 사용 중이면 다른 프로그램은 보통 같은 영역을 동시에 사용할 수 없습니다. 따라서 **자료구조와 알고리즘을 설계할 때 메모리 사용량은 중요한 조건**입니다. 예를 들어 알고리즘의 최대 메모리 사용량은 사용 가능한 메모리를 넘을 수 없습니다. 충분히 큰 연속 공간을 확보할 수 없다면 여러 곳에 나누어 저장할 수 있는 구조를 선택해야 합니다.

**물리 구조는 데이터가 컴퓨터 메모리에 저장되는 방식을 나타냅니다.** 기본 형태는 배열로 대표되는 연속 공간 저장과 연결 리스트로 대표되는 분산 공간 저장입니다. 저수준 저장 방식은 데이터의 접근, 갱신, 삽입, 삭제 방법에 직접 영향을 줍니다. 두 방식은 시간과 공간 측면에서 서로 다른 장단점을 가집니다.

![연속 공간 저장과 분산 공간 저장](classification_of_data_structure.assets/classification_phisical_structure.png)

모든 자료구조는 배열, 연결 리스트 또는 두 구조의 조합을 바탕으로 만들 수 있습니다.

- **배열 기반으로 구현할 수 있는 구조**: 스택, 큐, 해시 테이블, 트리, 힙, 그래프, 행렬, 3차원 이상의 텐서.
- **연결 리스트 기반으로 구현할 수 있는 구조**: 스택, 큐, 해시 테이블, 트리, 힙, 그래프.

연결 리스트는 초기화한 뒤에도 실행 중에 길이를 바꿀 수 있으므로 동적 자료구조라고 부릅니다. 배열의 길이는 초기화 뒤 고정되므로 배열은 보통 정적 자료구조로 봅니다. 하지만 더 큰 새 배열을 만들고 기존 데이터를 복사하면 제한적인 유연성을 얻을 수 있습니다.

> 물리 구조가 아직 잘 그려지지 않는다면 4장의 배열과 연결 리스트를 먼저 읽은 뒤 이 절로 돌아오세요.
