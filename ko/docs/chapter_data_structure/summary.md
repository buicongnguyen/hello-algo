# 3장 요약

### 핵심 복습

- 논리 구조는 원소 사이의 관계를 설명하고, 물리 구조는 데이터가 메모리에 놓이는 방식을 설명합니다.
- 배열, 연결 리스트, 스택, 큐는 선형 구조입니다. 트리, 힙, 그래프는 비선형 구조이며 해시 테이블은 여러 구조를 함께 사용할 수 있습니다.
- 각 메모리 영역에는 고유한 주소가 있고 프로그램은 이 주소를 이용해 데이터에 접근합니다.
- 기본적인 물리 저장 방식은 배열의 연속 공간과 연결 리스트의 분산 공간입니다.
- 기본 데이터형은 내용의 종류를 설명하고 자료구조는 내용을 조직하는 방법을 설명합니다.
- 부호 있는 정수는 보통 2의 보수를 사용하여 양수와 음수의 덧셈을 통일하고 두 가지 0 표현을 없앱니다.
- IEEE 754 `float`는 비트를 부호, 지수, 가수에 나눠 씁니다. 지수는 범위를 넓히지만 상대 정밀도를 낮춥니다.
- Unicode는 공통 코드 포인트 공간을 제공하고 UTF-8, UTF-16, UTF-32는 코드 포인트를 바이트로 인코딩합니다.
- UTF-8은 가변 길이이고 ASCII와 호환되며 파일과 네트워크 통신에 널리 사용됩니다.

### 질문과 답변

**해시 테이블이 선형 구조와 비선형 구조를 모두 포함할 수 있는 이유는 무엇인가요?**

해시 테이블의 기반은 보통 배열입니다. 충돌을 처리하기 위해 각 칸이 연결 리스트를 가리킬 수 있고, 리스트가 너무 길어지면 일부 구현은 이를 트리로 바꿉니다. 따라서 배열, 연결 리스트, 트리를 함께 사용할 수 있습니다.

**`char`는 언제나 1바이트인가요?**

아닙니다. 크기는 언어와 문자 표현 방식에 따라 달라집니다. C와 C++의 `char`는 보통 1바이트이고 UTF-16 코드 단위는 보통 2바이트입니다. Unicode 문자 하나가 여러 코드 단위를 필요로 할 수도 있습니다.

**배열 기반 구조를 ‘정적’이라고 부르는 것은 삽입과 삭제 연산과 모순되지 않나요?**

모순되지 않습니다. 연산은 동적으로 일어나도 기반 배열의 용량은 고정되어 있습니다. 용량을 넘으면 더 큰 배열을 할당하고 데이터를 복사해야 합니다.

**부호–절댓값 표현과 2의 보수 사이를 오갈 때 모두 ‘비트 반전 후 1 더하기’를 사용할 수 있는 이유는 무엇인가요?**

$n$비트 이진수에서 두 표현은 $2^n$에 대한 보수 관계입니다. 이 관계가 대칭이므로 비트를 반전하고 $1$을 더하는 같은 연산으로 서로 변환할 수 있습니다. $1$을 뺀 뒤 비트를 반전하는 것도 같은 과정을 다르게 쓴 것입니다.
