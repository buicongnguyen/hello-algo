# 5장 요약

### 핵심 복습

- 스택은 LIFO, 큐는 FIFO를 따르고 덱은 양쪽 끝에서 연산합니다.
- 스택의 `push`, `pop`, `peek`, 크기 확인은 모두 $O(1)$이어야 합니다.
- 큐는 뒤에 추가하고 앞에서 삭제하는 연산을 $O(1)$에 수행해야 하며 양끝 연결 리스트나 원형 배열로 구현할 수 있습니다.
- 덱은 스택과 큐를 모두 모방할 수 있습니다.
- 동적 배열은 캐시 지역성이 좋지만 가끔 확장하고, 연결 리스트는 유연하지만 참조 메모리가 필요합니다.
- 스택은 재귀, 실행 취소, 수식 분석, DFS, 백트래킹에 자주 사용됩니다.
- 큐는 BFS, 스케줄링, 버퍼, 요청 처리에 자주 사용됩니다.
- 덱은 슬라이딩 윈도, 0–1 BFS, 양끝 연산 문제에 유용합니다.

### 질문과 답변

**배열 앞을 스택 꼭대기로 사용하지 않는 이유는 무엇인가요?**

배열 앞에서 추가하거나 삭제하면 나머지 원소를 옮겨야 하므로 보통 $O(n)$입니다. 배열 끝을 사용하면 이동을 피할 수 있습니다.

**배열 큐는 삭제할 때마다 원소를 옮겨야 하나요?**

아닙니다. 원형 배열은 앞 인덱스만 증가시키고 모듈로 연산으로 빈 칸을 다시 사용합니다.

**덱이 스택과 큐를 완전히 대체할 수 있나요?**

기능상 가능하지만 더 좁은 인터페이스는 의도를 명확히 표현하고 알고리즘에서 허용하지 않는 연산을 막아 줍니다.
