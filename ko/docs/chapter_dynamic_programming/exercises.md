# 연습문제

## 개념 확인

### 동적 계획법은 언제 적합할까?

한 학습자가 “점화식을 쓸 수 있으면 언제나 동적 계획법을 사용해야 한다”고 말합니다.
아래 각 과제에 동적 계획법, 백트래킹, 또는 `dp` 표가 없는 반복문이나 수학 공식을 쓰는 것이 더 적절한지 판단하고 핵심 이유 하나를 설명하세요.

<!-- numbered-subquestions -->

1. 동전 단위 `[1, 3, 4]`를 반복해서 사용하여 금액 6을 가장 적은 동전으로 만드세요.
2. `[1, 2, 3]`의 모든 순열을 출력하세요.
3. $1 + 2 + \dots + n$을 계산하세요.

    동적 계획법이 적절하다고 판단한 과제에서는 `dp[i]`의 의미도 설명하세요.

??? success "정답"

    1. 동적 계획법이 적합합니다. `dp[i]`를 금액 `i`를 만드는 데 필요한 최소 동전 수라고 정의합니다.
        `i` 이하인 각 동전 `c`에 대해 `dp[i-c] + 1`을 후보로 두고 그중 최솟값을 선택합니다.
        서로 다른 선택 과정에서 같은 금액을 반복해서 만나며, 큰 금액의 최적해를 더 작은 금액의 최적해로 만들 수 있습니다.
        금액 6의 답은 `3 + 3`을 사용한 2개입니다.

    2. 백트래킹이 적합합니다. 이 과제는 6개의 순열을 하나씩 모두 생성해야 합니다. 백트래킹은 선택하고 탐색을 계속한 뒤,
        선택을 되돌리고 다른 가지를 시도할 수 있습니다. 어떤 방법을 사용하든 모든 순열을 실제로 출력하려면 전부 열거해야 합니다.

    3. 반복문이나 등차수열 합 공식이면 충분합니다. `S(i) = S(i-1) + i`라는 점화식을 쓸 수 있지만, `S(i)`는 더 작은 값 하나인 `S(i-1)`에만 의존합니다.
        각 부분 합은 한 번만 계산하므로 겹치는 하위 문제가 없고 `dp` 표도 필요하지 않습니다. “점화식을 쓸 수 있다”가 “동적 계획법이 필요하다”는 뜻은 아닙니다.

### 배낭 표의 한 칸 계산하기

물건 무게 `wgt = [1, 2, 3]`, 가치 `val = [5, 11, 15]`, 배낭 용량 4인 0-1 배낭 문제를 생각해 봅시다.
`dp[i][c]`는 앞의 $i$개 물건만 사용하고 배낭 용량 제한이 $c$일 때 얻을 수 있는 최대 가치입니다.
배낭을 정확히 가득 채울 필요는 없습니다.

`dp[3][4]` 상태만 계산하세요. `dp[2][4] = 16`, `dp[2][1] = 5`가 주어집니다.

<!-- numbered-subquestions -->

1. 세 번째 물건을 선택하지 않을 때 후보 가치는 얼마인가요?
2. 세 번째 물건을 선택하면 용량이 얼마나 남고 후보 가치는 얼마인가요?
3. `dp[3][4]`는 얼마여야 하나요? 이 값은 어떤 물건을 선택한 결과인가요?

??? success "정답"

    1. 세 번째 물건을 선택하지 않으면 앞의 두 물건으로 만든 결과를 유지합니다. 후보 값은 `dp[2][4] = 16`입니다.

    2. 세 번째 물건의 무게는 3이므로 배낭에 넣은 뒤 용량 $4-3=1$이 남습니다. 후보 값은
        `dp[2][1] + 15 = 5 + 15 = 20`입니다.

    3. 16과 20을 비교하면 `dp[3][4] = 20`입니다. 첫 번째와 세 번째 물건을 고른 결과이며,
        총무게는 $1+3=4$, 총가치는 $5+15=20$입니다.

        이 상태 계산은 0-1 배낭 문제에서 “선택하거나 선택하지 않기”를 비교하는 한 단계를 보여 줍니다.

### 배낭 용량은 어떤 순서로 갱신해야 할까?

0-1 배낭 문제에 무게 2, 가치 5인 물건 하나만 있고 배낭 용량은 4입니다.
물건은 최대 한 번만 선택할 수 있습니다. 초기 일차원 배열은 `dp = [0, 0, 0, 0, 0]`입니다.

한 학습자가 용량을 2에서 4까지 증가시키며 물건을 처리합니다.

- `dp[2]`를 갱신한 뒤 값은 5입니다.
- `dp[3]`을 갱신한 뒤 값도 5입니다.
- `dp[4]`를 갱신할 때 방금 얻은 `dp[2]`를 사용하여 `dp[4] = 10`을 만듭니다.

<!-- numbered-subquestions -->

1. `dp[4] = 10`은 올바른가요? 그 이유는 무엇인가요?
2. 각 물건을 최대 한 번만 선택할 수 있다면 `dp[4]`는 얼마여야 하나요?
3. 각 물건을 처리할 때 용량을 큰 값에서 작은 값으로 갱신해야 할까요, 반대로 해야 할까요? 어떤 문제를 막을 수 있나요?

??? success "정답"

    1. 올바르지 않습니다. 가치 10은 가치 5인 물건을 배낭에 두 번 넣은 것과 같아,
        각 물건을 최대 한 번만 선택할 수 있다는 조건을 위반합니다.

    2. 배낭에는 이 물건 하나만 넣을 수 있으므로 `dp[4]`의 올바른 값은 5입니다.

    3. 용량을 4, 3, 2처럼 큰 값에서 작은 값 순으로 갱신해야 합니다.
        그러면 `dp[c]`를 계산할 때 읽는 `dp[c-2]`가 현재 물건을 처리하기 전의 값으로 남아,
        같은 라운드에서 현재 물건을 재사용하지 않게 됩니다.

### 계단을 오르는 방법의 수

계단에는 `n`개의 칸이 있습니다. 한 번에 1칸 또는 2칸을 오르며 정확히 `n`번째 칸에 도착해야 합니다.
정상에 도달하는 서로 다른 방법의 수를 계산하세요. `n >= 1`이며, 방법은 1칸과 2칸 이동의 순서로 구분합니다.
일차원 동적 계획법 배열을 사용하세요. 지금은 두 상태만 유지하는 공간 최적화를 사용하지 않습니다.

??? tip "힌트"

    1. i번째 칸에 도착하는 마지막 이동은 1칸 또는 2칸뿐입니다
    2. 따라서 dp[i] = dp[i-1] + dp[i-2]입니다
    3. n이 1 또는 2인 경우를 먼저 처리한 뒤 3번째 칸부터 표를 채웁니다

[LeetCode](https://leetcode.com/problems/climbing-stairs/)

### 0-1 배낭

길이가 같은 배열 `wgt`와 `val`이 주어집니다. 물건 `i`는 양의 정수 무게 `wgt[i]`와 음이 아닌 정수 가치 `val[i]`를 가집니다.
배낭 용량 `cap`은 음이 아닌 정수입니다. 각 물건은 최대 한 번만 선택할 수 있습니다. `cap`을 넘지 않으면서
배낭에 담을 수 있는 최대 총가치를 일차원 동적 계획법으로 구하세요.

??? tip "힌트"

    1. 길이가 cap + 1인 배열 dp를 만들고 dp[c]를 용량 제한 c에서의 최대 가치로 정의합니다
    2. 물건 i를 처리할 때 선택하지 않는 dp[c]와 선택하는 dp[c-wgt[i]] + val[i]를 비교합니다
    3. 같은 라운드에서 현재 물건을 반복 선택하지 않도록 용량을 큰 값에서 작은 값 순으로 갱신합니다
