# 4장 요약

### 핵심 복습

- 배열과 연결 리스트는 연속 저장과 분산 저장을 대표하며 장단점이 서로 보완됩니다.
- 배열은 $O(1)$ 임의 접근과 낮은 구조 비용을 제공하지만 삽입·삭제는 보통 $O(n)$이고 길이가 고정됩니다.
- 연결 리스트는 위치를 알고 있을 때 연결만 바꾸어 $O(1)$에 삽입·삭제하고 유연하게 확장합니다. 대신 노드 접근은 $O(n)$이고 각 노드에 추가 메모리가 필요합니다.
- 대표적인 종류는 단일, 원형, 이중 연결 리스트입니다.
- 동적 리스트는 보통 동적 배열로 구현하는 순서 있는 집합입니다. 배열의 접근 장점을 유지하면서 용량을 자동으로 늘립니다.
- 동적 리스트 끝에 추가하는 연산의 분할 상환 비용은 $O(1)$이지만 한 번의 확장은 $O(n)$입니다.
- 디스크, RAM, 캐시는 속도, 용량, 비용 사이의 균형을 이루는 계층 구조입니다.
- 캐시 라인, 프리페치, 공간 지역성, 시간 지역성은 CPU가 RAM을 읽는 횟수를 줄입니다.
- 배열은 보통 연결 리스트보다 캐시 친화적이지만 최종 선택은 실제 요구 사항을 기준으로 해야 합니다.

### 질문과 답변

**스택에 놓인 배열과 힙 메모리에 놓인 배열의 효율은 같은가요?**

둘 다 원소를 연속해서 저장하므로 데이터 접근 효율은 비슷합니다. 하지만 스택 할당은 보통 빠르고 자동인 대신 용량이 작습니다. 힙 메모리는 더 크고 유연하지만 할당과 해제가 느릴 수 있고 단편화가 생기기 쉽습니다.

**배열 원소가 보통 같은 데이터형이어야 하는 이유는 무엇인가요?**

원소 크기가 같으면 다음 고정식으로 주소를 계산할 수 있습니다.

```text
원소 주소 = 배열 시작 주소 + 원소 크기 × 인덱스
```

원소마다 크기가 다르면 이 식만으로는 충분하지 않습니다. 연결 리스트는 참조를 따라 접근하므로 원칙적으로 노드마다 다른 종류의 데이터를 담을 수 있지만, 많은 구현은 타입 안전성을 위해 같은 데이터형을 사용합니다.

**노드 `P`를 삭제한 뒤 `P.next = None`으로 설정해야 하나요?**

머리 노드에서 더 이상 `P`에 도달할 수 없다면 알고리즘상 필수는 아닙니다. 그러나 복잡한 라이브러리나 자원 관리 시스템에서는 연결을 명시적으로 끊는 편이 더 안전하고 이해하기 쉽습니다.

**위치를 찾는 데 $O(n)$이 드는데 왜 연결 리스트 삽입과 삭제를 $O(1)$이라고 하나요?**

$O(1)$은 위치를 이미 알고 있을 때 연결을 바꾸는 연산만 설명합니다. 먼저 노드를 찾아야 한다면 전체 시간은 $O(n)$입니다. 덱처럼 양 끝의 참조를 유지하는 구조는 많은 삽입과 삭제를 실제로 $O(1)$에 수행할 수 있습니다.

**노드 그림의 참조 부분은 메모리를 얼마나 사용하나요?**

그림은 정성적인 설명일 뿐입니다. 값의 크기는 `int`, `long`, `double`, 객체 같은 데이터형에 따라 달라지고 참조 크기는 운영체제와 컴파일 환경에 따라 보통 4바이트 또는 8바이트입니다. 따라서 그림 속 블록 크기만으로 정확한 비율을 알 수 없습니다.

**동적 리스트 끝 추가는 항상 $O(1)$인가요?**

각 연산이 항상 그런 것은 아닙니다. 기반 배열이 가득 차면 $O(n)$ 할당과 복사가 필요합니다. 다만 용량을 일정 비율로 늘릴 때 여러 추가 연산의 분할 상환 비용은 $O(1)$입니다.

**동적 리스트에서 낭비되는 메모리는 어디에서 생기나요?**

리스트는 초기 용량을 갖고 $1.5$배나 $2$배 같은 비율로 확장합니다. 확장 직후에는 사용하지 않는 칸이 남습니다. 크기와 용량을 관리하는 변수에도 작은 비용이 있지만 빈 칸이 주된 낭비입니다.

**원소 객체가 연속해서 놓이지 않아도 Python 리스트는 배열인가요?**

그렇습니다. Python 리스트의 기반 배열은 객체 자체가 아니라 객체를 가리키는 참조를 연속해서 저장합니다. 그래서 실제 객체가 흩어져 있어도 인덱스의 참조를 $O(1)$에 얻을 수 있습니다.

**`[[0]] * n`은 서로 독립적인 하위 리스트 $n$개를 만드나요?**

아닙니다. 모든 원소가 같은 `[0]` 리스트를 가리킵니다. 독립된 리스트를 만들려면 리스트 컴프리헨션을 사용합니다.

```python
n = 4
independent = [[0] for _ in range(n)]
```

`[0] * n`에서도 여러 원소가 같은 작은 정수 객체를 참조할 수 있지만 정수는 불변입니다. 한 원소에 다시 값을 대입하면 그 위치의 참조만 바뀌므로 다른 위치는 변하지 않습니다.

**많은 C++ 알고리즘 문제에서 `std::list`를 직접 사용하지 않는 이유는 무엇인가요?**

`std::list`는 이중 연결 리스트라서 원소마다 포인터 두 개가 추가로 필요하고 데이터가 연속해서 놓이지 않습니다. 보통 `std::vector`보다 메모리를 많이 사용하고 캐시 활용도도 낮습니다. 따라서 동적 배열을 우선 사용하고 노드 단위 삽입·삭제의 이점이 정말 중요할 때 연결 리스트를 선택합니다.
