# 임의 접근 메모리와 캐시 *

배열과 연결 리스트는 연속 저장과 분산 저장이라는 두 물리 배치를 대표합니다. **물리 배치는 RAM과 캐시의 사용 효율에 큰 영향을 주며**, 결과적으로 프로그램 성능도 달라집니다.

## 저장 장치

컴퓨터에는 디스크, RAM, CPU 캐시라는 세 가지 중요한 저장 계층이 있습니다.

| 특성 | 디스크 | RAM | 캐시 |
| --- | --- | --- | --- |
| 목적 | 운영체제, 프로그램, 파일의 장기 저장 | 실행 중인 프로그램과 처리 중인 데이터의 임시 저장 | 자주 접근하는 데이터와 명령을 CPU 가까이에 저장 |
| 전원이 꺼질 때 | 데이터 유지 | 데이터 소실 | 데이터 소실 |
| 용량 | 크며 보통 TB 단위 | 더 작고 보통 GB 단위 | 매우 작고 보통 MB 단위 |
| 속도 | 가장 느림 | 빠름 | 가장 빠름 |
| GB당 비용 | 낮음 | 더 높음 | 매우 높고 보통 CPU에 포함됨 |

저장 계층은 피라미드처럼 생각할 수 있습니다. CPU에 가까운 계층일수록 빠르고 작으며 비쌉니다.

- 디스크를 RAM으로 완전히 대체할 수는 없습니다. RAM은 전원이 끊기면 데이터가 사라지고 훨씬 비쌉니다.
- 캐시는 동시에 매우 크고 매우 빠를 수 없습니다. 용량과 물리적 거리가 늘면 지연도 커집니다. L1, L2, L3 같은 여러 계층은 속도, 용량, 비용 사이의 균형입니다.

![컴퓨터 저장 계층 피라미드](ram_and_cache.assets/storage_pyramid.png)

> 저장 계층은 속도, 용량, 비용 사이의 대표적인 트레이드오프를 보여 줍니다. 비슷한 선택은 거의 모든 공학 시스템에 나타납니다.

디스크는 장기 데이터를 저장하고 RAM은 실행 중인 프로그램의 데이터를 보관하며 캐시는 자주 접근하는 데이터와 명령 일부를 저장합니다. 프로그램이 실행되면 데이터가 디스크에서 RAM으로 이동합니다. 캐시는 CPU가 더 빨리 접근할 수 있도록 RAM의 데이터를 미리 가져옵니다.

![디스크, RAM, 캐시, CPU 사이의 데이터 흐름](ram_and_cache.assets/computer_storage_devices.png)

## 자료구조의 메모리 효율

배열과 연결 리스트는 서로 다른 장점을 가집니다.

- 배열 원소는 서로 붙어 있고 연결을 저장하지 않아 구조 비용이 낮습니다. 하지만 충분히 큰 연속 공간이 필요하고 남는 용량이 생길 수 있으며 확장할 때 시간과 공간이 듭니다.
- 연결 리스트는 노드 단위로 할당하고 해제하므로 유연하지만 각 노드가 추가 연결을 저장해야 합니다.

메모리를 반복해서 할당하고 해제하면 빈 공간이 점점 조각납니다. 연속 저장 배열은 보통 단편화를 덜 일으키고, 흩어진 노드가 자주 바뀌는 연결 리스트는 단편화에 더 많이 기여할 수 있습니다.

## 캐시 효율

캐시는 RAM보다 작지만 훨씬 빠릅니다. CPU가 필요로 하는 데이터가 캐시에 없으면 **캐시 미스**가 발생하여 RAM에서 데이터를 가져와야 합니다. CPU가 캐시에서 바로 데이터를 찾는 비율을 **캐시 적중률**이라고 합니다.

캐시는 다음 메커니즘을 활용합니다.

- **캐시 라인**: 데이터는 개별 바이트가 아니라 일정 크기의 블록으로 전송됩니다.
- **프리페치**: 프로세서가 순차 접근이나 고정 간격 같은 패턴을 예측해 데이터를 미리 불러옵니다.
- **공간 지역성**: 한 위치에 접근하면 가까운 데이터도 곧 사용될 가능성이 큽니다.
- **시간 지역성**: 방금 사용한 데이터가 가까운 미래에 다시 사용될 가능성이 큽니다.

배열은 보통 연결 리스트보다 캐시를 잘 활용합니다.

- 연결 리스트 노드는 참조를 추가로 저장해 더 크므로 한 캐시 라인에 유용한 데이터가 적게 들어갑니다.
- 노드가 흩어져 있어 한 번 불러온 캐시 라인에 관련 없는 데이터가 많이 포함될 수 있습니다.
- 배열의 접근 패턴은 예측하기 쉽습니다.
- 배열의 연속 메모리는 공간 지역성이 좋습니다.

따라서 배열은 보통 캐시 적중률과 실제 성능이 더 좋습니다. 그렇다고 배열이 언제나 우수한 것은 아닙니다. 데이터가 매우 크고 자주 변하며 용량을 예측하기 어렵다면 연결 리스트 기반 구조가 큰 연속 공간 요구와 배열 확장 비용을 피할 수 있습니다.
