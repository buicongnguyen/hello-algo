# 탐색 알고리즘 다시 보기

탐색 알고리즘은 자료구조를 직접 순회하는 방법과 데이터의 구성 방식을 활용하는 방법으로 나눌 수 있습니다.

## 완전 탐색

선형 탐색, BFS, DFS는 전처리가 필요 없고 범용성이 높지만 대체로 $O(n)$ 시간이 필요합니다.

## 적응형 탐색

- 이진 탐색은 정렬된 배열을 활용해 $O(\log n)$에 탐색합니다.
- 해시 탐색은 해시 테이블로 평균 $O(1)$을 달성합니다.
- 트리 탐색은 균형 BST에서 $O(\log n)$입니다.

이러한 방법은 전처리나 추가 자료구조가 필요하고 데이터가 바뀔 때 유지 비용을 지불합니다.

## 방법 선택

- 한 번만 조회하거나 데이터가 작을 때: 선형 탐색.
- 크고 거의 바뀌지 않는 정렬 배열: 이진 탐색.
- 빠른 정확 일치 조회: 해시 테이블.
- 순서, 범위 조회, 갱신이 필요할 때: 균형 탐색 트리.
